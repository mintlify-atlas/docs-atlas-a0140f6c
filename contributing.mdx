# Contributing to Awesome Claude Code

We welcome contributions from the community! This guide explains how to recommend resources for inclusion in the Awesome Claude Code list.

## Recommendation Process

Recommending a resource is simple and handled through an automated system. The process is designed to be straightforward while maintaining the quality and consistency of the list.

<Steps>
  <Step title="Submit a Recommendation">
    Use the GitHub issue template to recommend a new resource. Visit the repository and click on "Recommend a new resource here" to open a pre-formatted issue.
    
    **Important**: Please do not open a pull request to submit recommendations. The only person allowed to submit PRs to this repository is Claude.
  </Step>
  
  <Step title="Provide Resource Details">
    Fill out the recommendation template with:
    - Resource name and description
    - GitHub repository URL
    - Category (Agent Skills, Workflows, Tooling, etc.)
    - Author information
    - Brief explanation of what makes this resource valuable
  </Step>
  
  <Step title="Automated Review">
    The automated system will process your recommendation and add it to the appropriate category if it meets the quality criteria.
  </Step>
</Steps>

### Quality Standards

Resources included in this list should:
- Be relevant to Claude Code development
- Provide clear documentation
- Be actively maintained (or provide significant standalone value)
- Follow best practices for their category
- Include proper licensing information

## Repository Enhancements

For suggestions about the repository itself (not resource recommendations), please [open a repository enhancement issue](https://github.com/hesreallyhim/awesome-claude-code/issues/new?template=repository-enhancement.yml).

## Categories for Resources

When recommending a resource, choose the most appropriate category:

### Agent Skills 🤖
Model-controlled configurations that enable specialized tasks

### Workflows & Knowledge Guides 🧠
Tightly coupled sets of Claude Code-native resources

### Tooling 🧰
Applications built on top of Claude Code (IDE integrations, monitors, orchestrators)

### Status Lines 📊
Status bar customizations and configurations

### Hooks 🪝
Scripts that run at different points in Claude's lifecycle

### Slash Commands 🔪
Custom prompts for specific tasks (Git, testing, documentation, etc.)

### CLAUDE.md Files 📂
Project configuration files (language-specific, domain-specific, scaffolding)

### Alternative Clients 📱
Alternative UIs and front-ends
