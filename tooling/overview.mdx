The Claude Code ecosystem includes a rich collection of tools and applications that extend its capabilities beyond slash-commands and `CLAUDE.md` files. These tools enhance productivity, provide better visibility into usage, enable multi-agent workflows, and improve configuration management.

## What Makes a Tool?

Tooling denotes applications that are built on top of Claude Code and consist of more components than simple slash-commands and `CLAUDE.md` files. These are typically:

- **Standalone applications** with their own CLI or GUI
- **Complex integrations** that connect Claude Code with other systems
- **Frameworks** that provide structured workflows and methodologies
- **Monitoring systems** that analyze and visualize Claude Code usage
- **Orchestration platforms** that manage multiple agents or tasks

## Getting Started

Explore each category to find tools that match your workflow:

- If you're new to Claude Code, start with [General Tools](/tooling/general) to enhance your basic workflow
- For editor integration, check [IDE Integrations](/tooling/ide-integrations)
- To track your usage and costs, visit [Usage Monitors](/tooling/usage-monitors)
- For complex multi-agent workflows, explore [Orchestrators](/tooling/orchestrators)
- To manage your configurations, see [Config Managers](/tooling/config-managers)
