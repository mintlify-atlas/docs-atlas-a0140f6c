Alternative Clients are alternative UIs and front-ends for interacting with Claude Code, either on mobile or on the desktop.

<CardGroup cols={2}>
  <Card title="Claudable" icon="window" href="https://github.com/opactorai/Claudable">
    Open-source web builder leveraging local CLI agents (Claude Code and Cursor Agent) to build and deploy products effortlessly.
    
    **By:** [Ethan Park](https://www.linkedin.com/in/seongil-park/)
  </Card>

  <Card title="claude-esp" icon="terminal" href="https://github.com/phiat/claude-esp">
    Go-based TUI that streams Claude Code's hidden output (thinking, tool calls, subagents) to a separate terminal. Watch multiple sessions simultaneously, filter by content type, and track background tasks.
    
    **By:** [phiat](https://github.com/phiat)
  </Card>

  <Card title="claude-tmux" icon="window-restore" href="https://github.com/nielsgroen/claude-tmux">
    Manage Claude Code within tmux. A tmux popup of all your Claude Code instances, enabling quick switching, status monitoring, session lifecycle management, with git worktree and pull request support.
    
    **By:** [Niels Groeneveld](https://github.com/nielsgroen)
  </Card>

  <Card title="crystal" icon="gem" href="https://github.com/stravu/crystal">
    A full-fledged desktop application for orchestrating, monitoring, and interacting with Claude Code agents.
    
    **By:** [stravu](https://github.com/stravu)
  </Card>

  <Card title="Omnara" icon="tower-control" href="https://github.com/omnara-ai/omnara">
    A command center for AI agents that syncs Claude Code sessions across terminal, web, and mobile. Allows for remote monitoring, human-in-the-loop interaction, and team collaboration.
    
    **By:** [Ishaan Sehgal](https://github.com/ishaansehgal99)
  </Card>
</CardGroup>

## Why Use Alternative Clients?

Alternative clients can enhance your Claude Code workflow by providing:

- **Multi-device access** - Work with Claude Code on mobile, web, or desktop
- **Enhanced visibility** - Monitor multiple sessions simultaneously
- **Team collaboration** - Share and collaborate on Claude Code sessions
- **Specialized interfaces** - TUI, GUI, or web-based interfaces for different preferences
- **Session management** - Better tools for managing multiple concurrent sessions

## Getting Started

Each client has different installation requirements and capabilities. Visit their GitHub repositories for detailed setup instructions and documentation.
