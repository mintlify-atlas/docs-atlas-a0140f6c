## Code Analysis Workflow

A typical code analysis workflow using these commands:

1. **Initial Check**: Run `/check` to perform comprehensive quality and security scanning
2. **Deep Analysis**: Use `/code_analysis` for detailed inspection and quality metrics
3. **Performance Review**: Apply `/optimize` to identify and fix performance bottlenecks
4. **Issue Reproduction**: Create test cases with `/repro-issue` for reported bugs
5. **TDD Implementation**: Use `/tdd` or `/tdd-implement` for new features

## Best Practices

When using code analysis and testing commands:

- **Run Checks Regularly**: Integrate `/check` into your workflow before commits
- **Follow TDD**: Use TDD commands for new features to ensure comprehensive test coverage
- **Document Issues**: Use `/repro-issue` to create clear, reproducible test cases
- **Optimize Strategically**: Focus optimization efforts on identified bottlenecks
- **Continuous Improvement**: Regularly review analysis results and adjust development practices
