## Context Loading Workflow

A recommended workflow for loading context:

1. **Initial Priming**: Start with `/context-prime` or `/prime` for overall project context
2. **Load Documentation**: Use `/initref` or `/load-llms-txt` for reference materials
3. **Domain Context**: Apply domain-specific commands like `/load_coo_context` as needed
4. **Refresh Context**: Use `/rsi` to reload context when switching between project areas

## Best Practices

When using context loading commands:

- **Prime Early**: Load context at the beginning of each session for best results
- **Be Selective**: Load only relevant context to avoid overwhelming Claude
- **Update Regularly**: Refresh context when project structure or conventions change
- **Combine Commands**: Use multiple context commands for comprehensive coverage
- **Document Patterns**: Create custom context commands for recurring project patterns
